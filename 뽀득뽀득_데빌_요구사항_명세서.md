## 1. 문서 개요

### 1.1 목적
본 문서는 아이부라보의 Flash(AS3) 게임 **"아슬아슬 창문닦기"** 의 게임플레이를 현대 웹 환경에 맞게 재구현하는 미니게임 **"뽀득뽀득 데빌"** 의 요구사항을 정의한다. 게임 메커니즘은 원작과 동일하게 유지하되, **모든 그래픽/사운드 리소스는 신규 악마 테마로 교체**한다.

### 1.2 참고 자료
- 원작: 아이부라보 "아슬아슬 창문닦기" (AS3 / Flash, `rankgame_41.waf`, 1.34MB)
- 원작 출처: https://vidkidz.tistory.com/7425 (와플래시 게임 아카이브)
- 원작 핵심 조작: 방향키 이동 + 스페이스바로 닦기

### 1.3 원작과의 차이
| 구분 | 원작 | 뽀득뽀득 데빌 |
|------|------|--------------|
| 기술 | Flash / AS3 | HTML5 Canvas + JS |
| 플랫폼 | PC (Flash Player) | PC + 모바일 (세로형 반응형) |
| 테마 | 달퐁마을 친구들 | 악마(데빌) 캐릭터 |
| 리소스 | 원작 에셋 | **신규 제작 (별도 업로드)** |
| 진행 | (정보 없음) | 5스테이지 + 엔딩 |

## 2. 게임 개요

### 2.1 컨셉
귀여운 꼬마 악마가 (지옥/고층 빌딩의) 창문을 **뽀득뽀득** 광이 나게 닦는 캐주얼 미니게임. 시간이 지나면 창문이 다시 더러워지고, 새/박쥐가 날아와 닦아 둔 창문을 다시 더럽힌다. 플레이어는 제한 시간 안에 목표 청결도를 달성해 스테이지를 클리어하고, 5스테이지를 모두 통과하면 엔딩을 본다.

### 2.2 핵심 재미 (Core Fun)
- **즉각적 손맛**: 더러운 면을 문지르면 즉시 깨끗해지는 시각적 쾌감 ("뽀득" 반짝임 연출)
- **공간 관리**: 새로 더러워지는 창문과 날아드는 새/박쥐 사이에서 동선을 효율적으로 관리
- **점진적 긴장감**: 스테이지가 오를수록 빨라지는 재오염 속도와 늘어나는 장애물

### 2.3 타겟
- 짧은 세션(1~3분)으로 즐기는 캐주얼 유저
- PC 웹 / 모바일 웹 양쪽 (세로형 우선)

## 3. 기술 사양

### 3.1 기술 스택 🟢
- **HTML5 Canvas 2D + 순수 JavaScript (Vanilla JS, 외부 게임 엔진 미사용)**
- 빌드/배포: 정적 파일(`index.html` + JS + 에셋)로 어디서나 호스팅 가능
- 의존성 최소화 (필요 시 경량 유틸만 사용)

### 3.2 플랫폼 / 조작 🟢
**PC + 모바일 모두 지원 (세로형 반응형)**

| 플랫폼 | 이동 | 닦기 |
|--------|------|------|
| PC | 방향키 (←↑↓→) | 스페이스바 (홀드) |
| 모바일 | **화면 가상 조이스틱** (좌측 하단) | **전용 닦기 버튼** (우측 하단, 홀드) |

- 캐릭터(스펀지)를 이동시킨 뒤, 닦기 입력을 유지하면 캐릭터가 위치한 타일을 닦는다. PC·모바일 메커니즘 동일.
- 🟡 가상 조이스틱/버튼 위치·크기는 세로형 한 손 플레이 기준으로 배치, 테스트 후 조정.

### 3.3 화면 / 반응형 🟢
- **기준 디자인 해상도: 720 × 1280 (세로형 9:16)**
- 캔버스는 화면 비율에 맞춰 스케일(letterbox), 논리 좌표계는 720×1280 고정
- PC: 중앙 정렬 + 좌우 여백(배경/장식). 모바일: 세로 풀스크린, 회전 불필요
- 🟡 안전영역(노치/홈바) 고려해 HUD·버튼은 가장자리에서 여유 패딩 확보

### 3.4 성능 목표 🟡
- 60 FPS 유지 (저사양 모바일에서 30 FPS 하한)
- 에셋 스프라이트 시트화로 드로우콜 최소화

## 4. 게임플레이

### 4.1 핵심 루프
```
타이틀 → 스테이지 인트로 → [이동 + 닦기 ↔ 창문 재오염/새·박쥐] → 목표 청결도 달성
   → 스테이지 클리어 → (다음 스테이지) ... → 5스테이지 클리어 → 엔딩 → 결과
                              └ 시간 초과 → 실패(재도전)
```

### 4.2 게임 목표 🟢 — 스테이지 클리어 + 엔딩
- 각 스테이지는 하나의 **창문(타일 격자)** 으로 구성
- **목표 청결도**(스테이지별 90~100%)에 도달하면 스테이지 클리어 → 다음 스테이지
- **총 5스테이지**, 마지막 스테이지 클리어 시 **엔딩** 연출 후 결과 화면
- 스테이지가 올라갈수록 점점 어려워짐(타일 수↑, 새/박쥐↑, 재오염 속도↑, 시간↓)

### 4.3 닦기 메커니즘 🟡
- 창문 표면은 다수의 **타일(pane)** 로 분할되며, 각 타일은 `오염도 0~100%`를 가짐
- 캐릭터가 타일 위에서 닦기 입력을 유지하면 해당 타일 오염도가 감소
- 오염도 0% 도달 시 "뽀득" 반짝임 이펙트 + 효과음
- 전체 청결도 = (전체 타일 청결량 합) / (전체 타일 수)

### 4.4 재오염 (시간 경과) 🟡
- 일정 주기로 무작위 타일의 오염도가 상승 (스테이지별 속도 상이)
- 이미 닦은 면도 방치하면 다시 더러워져 동선 관리가 핵심

### 4.5 장애물 🟢 — 새/박쥐 (재오염)
- **새/박쥐**가 화면을 가로질러 날아다니며, 지나간 경로의 타일을 다시 더럽힌다.
- 스테이지가 오를수록 등장 수·속도 증가.
- 캐릭터와 접촉 시 페널티 발생 (§4.6).
- 🟡 1차 버전은 새/박쥐 단일 장애물로 구성. (비/끈적이/벌레 등은 향후 확장 후보로 보류)

### 4.6 클리어 / 실패 조건 🟢 — 혼합 (시간제 + 접촉 페널티)
- **클리어**: 제한 시간 내 목표 청결도 달성
- **실패**: 제한 시간 초과 시까지 목표 미달 → 스테이지 실패(재도전)
- **접촉 페널티**: 새/박쥐에 부딪히면 페널티 발생
  - 🟡 제안 수치: 남은 시간 **-3초** + 짧은 이동 둔화(약 1초) + 피격 모션/효과음
- 라이프(목숨) 개념은 사용하지 않으며, 압박은 전적으로 "시간"으로 표현

### 4.7 점수 시스템 🟢/🟡 — 랭킹 없음, 점수 표시만
- 🟢 별도 랭킹/서버 연동 **없음**. 점수는 인게임 HUD와 결과 화면에 표시.
- 🟡 점수 구성 제안:
  - 닦은 타일 수 × 기본 점수
  - 빠른 클리어 **시간 보너스**
  - **콤보**: 끊김 없이 연속으로 타일을 깨끗이 만들면 배수 증가
  - 스테이지 클리어 보너스
- 🟡 최고기록 저장은 범위 외(미저장). 세션 내 누적 점수만 표시.

## 5. 화면 구성 / UI

### 5.1 화면 흐름
```
[타이틀] → [튜토리얼(최초 1회/선택)] → [스테이지 인트로] → [인게임]
   → [스테이지 클리어] → 다음 스테이지 / (5스테이지 후) [엔딩] → [게임 결과] → 타이틀
                                                              └ 실패 → 결과(재도전)
```

### 5.2 화면별 요소 (세로형 720×1280 기준)
- **타이틀**: 로고("뽀득뽀득 데빌"), 시작 버튼, 사운드 토글
- **인게임 HUD**:
  - 상단: 현재 스테이지 / 남은 시간 / 전체 청결도 게이지 / 점수
  - 중앙: 창문(타일 격자) + 악마 캐릭터 + 새/박쥐
  - 하단: 좌측 가상 조이스틱, 우측 닦기 버튼 (모바일). PC는 비표시
- **스테이지 클리어**: 획득 점수, 시간 보너스, 다음 버튼
- **엔딩**: 5스테이지 클리어 축하 연출
- **게임 결과**: 총점, 다시하기 / 타이틀 버튼

### 5.3 연출 🟡
- 닦기 "뽀득" 반짝임, 물거품/스펀지 흔적
- 캐릭터 애니메이션: 대기 / 이동 / 닦기 / 피격
- 스테이지 클리어 시 창문 광택 + 악마 만족 모션

## 부록 A. 리소스 명명 규칙 & 규격 가이드 🟢

신규 에셋을 매끄럽게 교체할 수 있도록 아래 규칙을 권장한다. (개발 코드에서 동일 키로 로드)

### A.1 폴더 구조
```
assets/
├─ character/   캐릭터 스프라이트
├─ window/      창문 타일·프레임
├─ obstacle/    새/박쥐 등 장애물
├─ effect/      이펙트(반짝임·물거품)
├─ background/  배경
├─ ui/          버튼·게이지·아이콘·로고
├─ sfx/         효과음
└─ bgm/         배경음악
```

### A.2 명명 규칙
- **소문자 + 스네이크 케이스**, 형식: `분류_이름_상태_번호`
- 연속 프레임은 2자리 0패딩 번호: `_01`, `_02` …
- 스프라이트 시트는 `*_sheet.png` + 동명 `*_sheet.json`(프레임 좌표) 권장

### A.3 규격 권장
- **기준 해상도**: 720×1280(@1x). 선명도를 위해 **2x(1440×2560)로 제작 후 다운스케일** 권장
- **이미지 포맷**: PNG, 투명 배경(32bit). 사진성 배경만 JPG 허용
- **창문 타일**: 정사각 권장(예: 1x 기준 한 변 120~160px). 오염 단계는 3단계(약함/중간/심함) 기준
- **캐릭터**: 1x 기준 가로/세로 약 200~256px 내. 프레임 크기 균등(앵커 동일)
- **장애물(새/박쥐)**: 1x 기준 약 96~128px, 날갯짓 3~4프레임
- **사운드**: `mp3`(또는 `ogg`), 모노 권장 SFX / 스테레오 BGM, -14 LUFS 내외 노멀라이즈
- **스프라이트 시트**: 프레임 균등 간격, 여백 2px, 시트당 단일 캐릭터/객체

> 사내에 기존 네이밍/규격 규칙이 있다면 알려주시면 본 가이드를 그에 맞춰 갱신합니다.
