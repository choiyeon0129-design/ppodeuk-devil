# 뽀득뽀득 데빌 (Window Washer)

악마 캐릭터가 창문을 닦는 캐주얼 미니게임. HTML5 Canvas + 순수 JavaScript.

- 기획: [뽀득뽀득_데빌_요구사항_명세서.md](뽀득뽀득_데빌_요구사항_명세서.md)
- 현재 단계: **M1 — 코어 프로토타입** (이동 · 닦기 · 청결도 판정 · 스테이지 클리어/실패)

## 실행 방법
- 가장 간단: `index.html` 더블클릭 (브라우저에서 바로 실행)
- 또는 로컬 서버:
  ```
  python -m http.server 8000
  # http://localhost:8000 접속
  ```

## 조작
화면 버튼 전용 (PC 키보드 미사용 — 마우스/터치 동일):

| 이동 | 닦기 |
|------|------|
| 좌측 하단 방향 버튼(D-pad), 한 칸씩 (꾹 누르면 연속) | 우측 하단 "닦기" 버튼 — **창문당 5번** |

타이틀/결과 화면에서는 아무 곳이나 탭/클릭으로 진행.

## 규칙 (M1)
- 모든 창문 타일이 더러운 상태로 시작 → 닦아서 **목표 청결도** 달성 시 스테이지 클리어
- 제한 시간 초과 시 실패(재도전)
- 총 5스테이지, 마지막 클리어 시 엔딩
- 점수 = 깨끗하게 만든 타일 × 100 + 클리어 시 남은 시간 보너스

## 파일 구조
```
index.html          진입 + 캔버스
style.css           9:16 letterbox 스케일
js/config.js        설정 / 스테이지 데이터 / 유틸
js/input.js         키보드 + 가상 조이스틱/버튼 입력
js/grid.js          창문 타일·오염도·청결도·반짝임
js/player.js        캐릭터 이동/닦기 모션
js/game.js          상태머신 + 루프 + 렌더(HUD/오버레이)
js/main.js          진입점
cha_1.png           캐릭터 리소스(placeholder)
```

## 미구현 (다음 단계)
- **M2**: 시간 경과 재오염(`stage.redirt`), 사운드, 타이틀/결과 폴리싱
- **M3**: 새/박쥐 장애물 + 접촉 페널티(-3초·둔화), 5스테이지 밸런싱, 엔딩 연출
- **M4**: 신규 리소스 교체 (부록 A 명명 규칙), 애니메이션/이펙트
- **M5**: 모바일 최적화, 성능, QA

> 리소스는 요구사항 명세서 **부록 A**의 명명 규칙(`char_idle.png`, `window_dirty_01.png` 등)에 맞춰 `assets/` 폴더에 업로드하면 코드에서 교체 연동됩니다.
